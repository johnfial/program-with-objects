Stage 3: Object Oriented Programming

Abstraction
  • Python includes many programs in its "standard library," such as "webbrowser" and "time". This hiding of details is called abstraction. Abstraction helps because we can access functions that have already been programmed and explained in detail in documentation, allowing programmers to focus on the new programming necessary to accomplish their goals.
  • What we've done so far is use a library file (webbrowser, os, etc.) and a function (webbrowser.open or os.rename). Turtle contains something different: a class.
Classes
  • A class is a "neatly packaged box" that puts things together quite well. Lets us write turtle.Turtle() which actually calls the initialize function. In the instructor's analogy, he equates the term class to a blueprint of a building. A class can have both data (instance variables) and methods (instance methods).
    • The word instance (aka object) is used exactly like I'm used to from general computer usage and gaming. In the blueprint analogy, it's a single object based on the class blueprint.
    • When we call the class (such as turtle), we're really calling the init function within the class to initialize a new instance.
    • Instance variables are the individual variables (like movie title, storyline, etc.) within an instance.
    • An instance method is a function that is defined inside of a class.
    • Classes are the core of Object Oriented Programming (OOP). An object is an instance of a class, and is where the name comes from.
    • Is something an object or a class? Remember the question: Is it a salmon or a fish? Or, is it a fish, a salmon, or a salmon named Mary? See this page if confused.
    • Examples:
  • Class Variables
    • A class variable is a higher-level variable, defined at the level of the class itself, before the init function and its instance variables are initialized. For example, remember that we used the movie ratings (G, PG, etc.) as a class variable, since those wouldn't change depending on the movie. How to tell? Is the variable more associated with the class as a whole, or each instance in general? Asking that question should help.
    • One predefined class variable is "doc", which is accessed by filename.Classname.__doc__ with two underscores. It is set by being the first line in the class code with triple quotes.
    • Other predefined class variables can be found at this page. Note that "name" and "module" call their appropriate names. Module refers to the filename loaded (the "media" in media.py, for example), and name refers to the name of the class.
  • Classes: Style and Misc. Notes:
    • Note that the Google Style Guide says it's good practice to have the first letter of a class name in Caps.
    • It's good practice to define your class in one file, and REFERENCE that class from another (working) file.
    • self is not technically a python keyword, but a useful convention used by programmers, which makes it easier to read code.
Misc. Notes
  • A constructor is called whenever we create an instance of a class. The constructor is basically the "init" method inside the class, which uses the "self" (instance) name to initialize all the instance variables.
  • Inheritance
    • Inheritance is where child objects inherit attributes, methods, etc. from their parents. We were already doing this in Modules 1 and 2, where HTML "objects" had a certain class whose style could be defined in the adjacent CSS file. That way, once we edit the CSS, the changes apply to all the applicable HTML objects of that class. Same thing in python: any child classes have the attributes of their parent classes, as well, allowing us to re-use code. When in doubt, review the inheritance.py file in lesson 3.4b.advanced in order to understand child and parent class attributes. Before running the code, mentally figure out what will print first!
    • When both the child and parent have the same attribute or method (function), and it is called on a child class, the code in the child's class will be executed (so if the child "show_info" function had three print statements, and the parent's only had two, child.show_info() would print three statements). This is called method overriding.
  • The difference between from as opposed to import: the former only imports the specified functions or classes within the library/file specified, whereas the latter imports the entire library. In the latter case, functions need to be referenced by using libraryname.function, but in the former case, referencing function will suffice. Note that there are pros and cons to each!

Functions (Also called procedures)

A function is a "task" that is performed when executed, and can be repeated multiple times. Rather than simply performing a task on a print line or with an individual command in the interpreter, for example, functions can be called time and time again to perform their task. Functions can contain other functions, however...

Functions should be divided into their smallest useful parts, this way, it's easier to re-use individual functions without them being buried within other functions.

There are many defined functions within python, but a new one is defined by writing def function_name(input1,input2): followed by the function's code.

Inputs to functions can also be called operands or arguments.

Lesson 1.3: CSS

CSS stands for Cascading Style Sheets, used for styling.

The MOST specific rule applies always (this is what cascading means).

Under the hood, everything in both HTML and CSS are rectangular shapes (even if they look otherwise).

An HTML HEAD tag attribute called "rel" with value "stylesheet" points to the style.css file, which tells the page how to style all its elements based on that .css file.

Lesson 1.1: HTML

An element is the entire structure of opening, main, and closing tags.

Remember to always close tags as soon as you write them!

The different variables of a tag are called "attributes," such as the "src" or "alt" attributes.

<a> is called the anchor tag.

The image tag is an example of a "Void" tag: it has no content, and doesn't need an end tag.

Tags can be inline tags (such as a break tag) or block tags (such as a paragraph tag). SPAN is an inline tag, but DIV is a block tag.
